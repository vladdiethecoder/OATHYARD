# Large-File Policy for OATHYARD

**Date:** 2026-07-02
**Status:** Active

## Files exceeding 50 MiB

| File | Size | Decision | Rationale |
|------|------|----------|-----------|
| `assets_src/production/meshy_seed/gambeson.obj` | 89 MB | Runtime asset replaces source OBJ | The decimated runtime mesh (`assets/runtime/seed/gambeson.mesh.json`, ~4.7 MB) is used by the renderer. The original OBJ is source provenance only. SHA256 is recorded in the mesh JSON. |

## Policy

1. Source GLB files (Meshy-6 exports) are kept as provenance evidence; they are typically 5-20 MB.
2. Runtime meshes are generated by Blender decimation to ~30k vertices and exported as JSON (~5 MB each).
3. No runtime path reads the OBJ files; they exist for provenance/hash verification only.
4. Files over 50 MiB that are not needed at runtime are candidates for Git LFS or `.gitignore` exclusion.
5. The `gambeson.obj` file is retained in the repo for now but should be migrated to Git LFS before public release.
6. Any future generated assets exceeding 50 MiB must be either:
   - Converted to a validated runtime asset < 50 MiB before commit
   - Added to Git LFS
   - Or explicitly allowlisted in this document with a decision record

## Verification

```sh
find assets assets_src content -type f -size +50M -print
```

Expected output: `assets_src/production/meshy_seed/gambeson.obj` (allowlisted above).
