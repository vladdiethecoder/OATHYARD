# OATHYARD artifact and Kanban hygiene

Purpose: keep the current renderer / physical-fidelity lane visible while old generated evidence remains available but stops dominating the repo.

This workflow is intentionally fail-closed:

- It never deletes artifacts.
- Default mode only writes current reports under `artifacts/current/`.
- Archive mode moves stale top-level artifact directories/files to `artifacts/_stale/<UTC>/` and writes an `archive_manifest.json` mapping original path to archive path.
- Active Kanban task artifacts are protected.
- Canonical evidence and gate directories (`kanban`, `verify_a`, `verify_b`, `final`, `final_gates`, `readiness`, `secrets`, `truth_*`, `negative_audit`, `runtime_3d`, package evidence, etc.) are protected by default.

## Current entrypoint

Open this first:

```sh
artifacts/current/LATEST.md
```

It is regenerated by:

```sh
./tools/oathyard_hygiene.sh --out artifacts/current --min-age-hours 24
```

Outputs:

- `artifacts/current/LATEST.md` — human current-state report.
- `artifacts/current/kanban_focus.md` — same current-state report, named for quick grep/opening.
- `artifacts/current/kanban_focus.json` — machine-readable Kanban/artifact focus payload.
- `artifacts/current/artifact_inventory.json` — top-level artifact inventory with sizes, ages, and classification reasons.
- `artifacts/current/stale_artifacts.md` — stale candidates and archive commands.

## Non-destructive archive mode

Archive only stale candidates older than the retention window:

```sh
./tools/oathyard_hygiene.sh \
  --out artifacts/current \
  --min-age-hours 72 \
  --archive \
  --max-move 20
```

Archive mode moves candidates to:

```text
artifacts/_stale/<UTC>/<artifact-name>
```

Each archive run writes:

```text
artifacts/_stale/<UTC>/archive_manifest.json
```

If an old task summary points at an original artifact path, consult the archive manifest before assuming the evidence is missing.

## Kanban guardrail

The current ordering is:

1. physical-fidelity gap/research;
2. deterministic reduced biomechanics/tissue/material architecture;
3. minimal truth spike and asset metadata contract;
4. only then Bevy/Unreal renderer/import/capture work;
5. only after that visual benchmark / owner acceptance / readiness gates.

Renderer-only, import-only, capture-only, non-native frame/non-native diagram/native-status/software/debug, or scalar-table evidence must not close high-fidelity cards.

## Continuous local automation

Two local Hermes cron jobs maintain this lane:

- `OATHYARD artifact/Kanban hygiene`: script-only, every 2h. Refreshes `artifacts/current/` and non-destructively archives old generated candidates under `artifacts/_stale/<UTC>/` after the retention window.
- `OATHYARD Kanban steward`: agent-driven, every 4h. Re-runs the hygiene report, inspects Kanban diagnostics and running/ready queues, and parks/reclaims renderer/import/capture/readiness drift that bypasses physical-fidelity prerequisites.

CLI-origin cron delivery is local-only; inspect jobs with:

```sh
hermes cron list
```

Cron should stay quiet when diagnostics are clean and no archive moves occur. It still refreshes `artifacts/current/` every run.
